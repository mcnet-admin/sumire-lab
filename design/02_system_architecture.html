各機能は共通データを利用することで、 同じ内容を何度も入力することなく 運用できるよう設計しています。

🐙 GitHub構成

本プロジェクトでは、 GitHub上で役割を分けて管理しています。 


GitHub

├── sumire-portal
│
│   実際に運用する予約システム
│
└── sumire-lab
    開発室
    設計書
    マニュアル
    更新履歴

sumire-portal は実際に運用するシステム、 sumire-lab は 設計・記録・引継ぎのための開発室です。

✨ システムの特徴

  • 園児情報を一元管理
  • 予約から請求までデータを連携
  • GitHubによるバージョン管理
  • 継続的な改善を前提とした設計
  • 将来担当者への引継ぎを重視

📁 フォルダー構成

システム本体と開発室は、それぞれ目的に応じたフォルダー構成で管理しています。 


sumire-portal/
│
├── css/
├── js/
├── data/
├── templates/
├── index.html
└── ...

sumire-lab/
│
├── css/
├── design/
├── manual/
├── history/
├── img/
├── index.html
└── README.md

このように役割を分けることで、 システム本体とドキュメントを独立して管理できるようにしています。

🛠️ 使用技術

技術 主な用途
HTML 画面表示・ドキュメント作成
CSS デザイン・レイアウト
JavaScript 予約画面の動作
Python データ処理・請求書作成
GitHub ソースコード管理
GitHub Pages 開発室の公開
GitHub Actions 自動処理・自動更新
Google Workspace メール・各種サービス連携
